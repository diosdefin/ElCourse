### Стратегия использования квот GitHub Copilot
* **Бесплатные модели (GPT-4o / GPT-4.1 / Claude 3.5 Sonnet):** Используй для UI/UX, базового CRUD, верстки карточек, кнопок и базовой логики Vue.
* **Премиум модели (Codex 5.3 / Gemini 3.1 Pro):** Трать свои 300 запросов **ТОЛЬКО** на сложную архитектуру (WebSockets, Celery, FFmpeg, миграция токенов, сложные SQL-запросы). 
Это исторический момент. Мы собрали все твои идеи, бизнес-логику, геймификацию, безопасность и фишки современных платформ в единый, монолитный план. 

Ниже представлен **Финальный Мастер-План Промтов**. Сохрани этот текст в отдельный файл. Копируй эти промты строго по очереди и скармливай их GitHub Copilot. Я уже распределил их по моделям, чтобы ты не сжег премиум-лимиты на простые задачи.

### 📦 ЭПИК 1: Умная авторизация и Профиль (Модель: GPT-4o / GPT-4.1 — Бесплатная)

**Промт 1.1: Аутентификация, OAuth2 и Сброс пароля**
> "Реализуй современную систему аутентификации во Vue 3 и Django DRF. 
> 1. На бэкенде: установи `dj-rest-auth` и `django-allauth`. Настрой стандартную регистрацию, логин (выдача JWT), и эндпойнты для сброса пароля (Password Reset через отправку Email). Настрой Social Login для входа через Google и GitHub.
> 2. Во фронтенде: Создай компонент `AuthView.vue` (стеклянный морфизм). Сделай вкладки Логин/Регистрация. Регистрация должна быть пошаговой (Шаг 1: Данные, Шаг 2: Выбор роли карточками). Добавь кнопки 'Войти через Google/GitHub'. 
> 3. Реализуй форму 'Забыл пароль?' и страницу `ResetPasswordConfirm.vue`, которая принимает токен из URL и задает новый пароль."

**Промт 1.2: Настройки профиля**
> "Создай страницу `/settings` во Vue 3. Раздели её на 3 вкладки (Tabs): 
> - 'Основные': загрузка аватара (через FormData с превью), редактирование bio.
> - 'Контакты': добавление ссылок на Telegram, LinkedIn, GitHub.
> - 'Безопасность': форма смены пароля.
> Напиши соответствующие PATCH эндпоинты в DRF (`/api/profile/update/`). Добавь красивые toast-уведомления (успех/ошибка) после сохранения."

### 💼 ЭПИК 2: Биржа труда (Модель: GPT-4o / GPT-4.1 — Бесплатная)

**Промт 2.1: Вакансии и Заявки**
> "Создай функционал 'Биржи' (Job Board) для мэтчинга работодателей и студентов.
> 1. В `models.py` создай модель `Vacancy` (работодатель, заголовок, описание, бюджет, требуемые навыки [M2M из Skill]) и `StudentPitch` (студент, заголовок, ставка в час, доступность в часах, описание).

> 3. Во Vue 3 создай `JobsView.vue`. Сделай два таба: 'Вакансии' (видят студенты) и 'Кандидаты/Резюме' (видят работодатели). В карточках выведи теги навыков. Добавь кнопку 'Связаться', которая создает сущность `JobOffer`."

### 🏗️ ЭПИК 3: Архитектура Курсов и Конструктор Учителя (Модель: GPT-5.3 Codex / Gemini 3.1 Pro — ПРЕМИУМ)

**Промт 3.1: Новая БД Курсов и Модули**
> "Полностью обнови архитектуру курсов в Django.
> 1. Новая иерархия: `Course` -> `Module` (order, title) -> `Lesson`.
> 2. Модель `Lesson` теперь имеет поле `type` (Video, Text, Quiz, FinalExam, PracticalTask).
> 3. Добавь модель `LessonAttachment` (файл, название) с ForeignKey на Lesson.
> 4. Для `Quiz` добавь настройки по умолчанию: `time_limit_seconds` (по умолч. 30 на вопрос), `passing_score_percentage` (80), `max_attempts` (3).
> Напиши миграции и обнови сериализаторы."

**Промт 3.2: Идеальный конструктор для преподавателя**
> "Создай компонент `CourseBuilder.vue` во Vue 3. 
> Интерфейс: минималистичный drag-and-drop список модулей и уроков. При добавлении урока можно прикрепить файлы (документы/фото). Рядом с уроками типа Quiz добавь иконку ⚙️ (Настройки). При клике открывается модалка: 'Проходной балл', 'Лимит времени на вопрос', 'Макс. попыток' и 'Штрафной таймаут (в часах)'. Реализуй UI для создания типа урока 'Final Exam' (Босс-тест в конце курса) и 'PracticalTask' (где студент должен будет сдать ссылку на GitHub)."

### 🎓 ЭПИК 4: Прохождение курса и Геймификация (Модель: GPT-5.3 Codex / Gemini 3.1 Pro — ПРЕМИУМ)

**Промт 4.1: Жесткая логика прохождения (Duolingo Style)**
> "Реализуй бэкенд и фронтенд логику строгого прохождения курса (`CoursePlayView.vue`).
> 1. Модули и Уроки открываются СТРОГО последовательно. Ученик не может открыть Модуль 2, пока не сдаст тесты Модуля 1.
> 2. Логика Quiz: Сверху плавающий таймер. Вопросы по одному. Если студент проваливает тест 3 раза (или сколько указал учитель), создается блокировка в `QuizAttempt` на 24 часа. Покажи студенту алерт: 'Попытки исчерпаны. Тест заблокирован на 24 часа. Повторите материал'.
> 3. Босс-Файт: Урок типа `FinalExam` генерирует вопросы из всех модулей курса.
> 4. Практика: Урок типа `PracticalTask` выводит поле для ввода ссылки на GitHub/загрузки файла. Статус урока становится 'На проверке', пока учитель не нажмет Approve/Reject через свой дашборд."

### 🎬 ЭПИК 5: HLS Видео Инфраструктура (Модель: GPT-5.3 Codex / Gemini 3.1 Pro — ПРЕМИУМ)

**Промт 5.1: Конвертация и Умный Плеер**
> "Реализуй HLS видео-пайплайн и плеер с Resume Watching.
> 1. Бэкенд: Настрой Celery таску `convert_to_hls` (через `ffmpeg-python`). При загрузке `mp4` режь его на `.ts` сегменты и генерируй `.m3u8` (1080p, 720p, 480p).
> 2. Фронтенд: В `LessonView.vue` создай кастомный плеер на базе `hls.js`. Добавь переключатель качества.
> 3. Логика: Плеер каждые 10 секунд делает POST на `/api/lessons/{id}/progress/` с текущим таймкодом. При открытии урока видео автоматически начинается с сохраненного места."

### 🧹 ЭПИК 7: Финальная полировка и Безопасность (Модель: GPT-5.3 Codex / Gemini 3.1 Pro — ПРЕМИУМ)

**Промт 7.1: Миграция Токенов на HttpOnly Cookies**
> "Перепиши хранение JWT токенов для безопасности. 
> Сейчас токены лежат в `localStorage`. Настрой DRF SimpleJWT на выдачу HttpOnly cookies (`JWT_AUTH_COOKIE`, `JWT_AUTH_HTTPONLY = True`, `CORS_ALLOW_CREDENTIALS = True`). Во фронтенде (`api.js`) удали ручное добавление заголовка `Authorization`, включи `withCredentials: true`. Напиши axios-интерцептор для автоматического silent-обновления токена (refresh) при получении 401 ошибки."

--------------------------------------------------------

**Промт 7.2: Глобальная очистка и Оптимизация (ДАВАТЬ В САМОМ КОНЦЕ)**
> "Сделай финальный рефакторинг всего проекта для продакшена.
> 1. Бэкенд: Проанализируй все ViewSets. Избавься от N+1 запросов, добавив `select_related` и `prefetch_related`. Настрой Redis кэширование для публичных эндпоинтов (список курсов, сообщество).
> 2. Удали все 'мусорные' принты (`print()`, `console.log()`), неиспользуемые переменные и закомментированный старый код.
> 3. Приведи код к стандарту PEP8, добавь docstrings к сложным функциям. Убедись, что все лоадеры во Vue работают корректно при долгих запросах."
