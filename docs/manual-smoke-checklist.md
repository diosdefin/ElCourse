# ElCourse Manual Smoke Checklist

Дата проверки: __________
Проверяющий: __________
Backend URL: __________
Frontend URL: __________

## Подготовка

- [ ] Backend запущен без ошибок.
- [ ] Frontend запущен без ошибок.
- [ ] Есть тестовые аккаунты: student, teacher, employer.
- [ ] В браузере открыты DevTools Console и Network.
- [ ] На каждом сценарии нет красных ошибок в Console.

## Guest

- [ ] Гость открывает главную страницу.
- [ ] Гость открывает каталог курсов `/courses`.
- [ ] Гость открывает опубликованные вакансии `/vacancies`.
- [ ] Гость открывает публичный профиль пользователя.
- [ ] Гость при переходе на урок перенаправляется на login.
- [ ] Гость при попытке откликнуться на вакансию перенаправляется на login или получает понятное сообщение.

## Student

- [ ] Student регистрируется и попадает в свой раздел.
- [ ] Student входит через login.
- [ ] Student видит свой профиль `/profile`.
- [ ] Student редактирует профиль и настройки `/settings`.
- [ ] Student открывает каталог курсов.
- [ ] Student открывает курс и видит прогресс.
- [ ] Student открывает доступный урок.
- [ ] Student не открывает заблокированный или неопубликованный урок.
- [ ] Student проходит текстовый или видеоурок.
- [ ] Student открывает quiz/final exam, отправляет ответы и видит результат.
- [ ] Student открывает вакансии.
- [ ] Student откликается на опубликованную вакансию.
- [ ] Student видит свои отклики.
- [ ] Student видит уведомления по offer/application.

## Teacher

- [ ] Teacher регистрируется и попадает в кабинет автора.
- [ ] Teacher входит через login.
- [ ] Teacher открывает `/teacher`.
- [ ] Teacher создаёт курс.
- [ ] Teacher открывает конструктор курса.
- [ ] Teacher создаёт модуль.
- [ ] Teacher создаёт урок.
- [ ] Teacher создаёт quiz-вопросы.
- [ ] Teacher открывает preview своего урока.
- [ ] Teacher не может редактировать чужой курс.
- [ ] Teacher открывает аналитику `/teacher/analytics`.
- [ ] Teacher не видит employer-only actions.

## Employer

- [ ] Employer регистрируется и попадает в раздел кандидатов `/employer`.
- [ ] Employer входит через login.
- [ ] Employer ищет студентов.
- [ ] Employer открывает карточку студента.
- [ ] Employer отправляет offer студенту.
- [ ] Employer открывает вакансии `/vacancies`.
- [ ] Employer создаёт вакансию.
- [ ] Employer редактирует свою вакансию.
- [ ] Employer закрывает свою вакансию.
- [ ] Employer видит отклики на свои вакансии.
- [ ] Employer меняет статус отклика.
- [ ] Employer не открывает teacher endpoints.

## Courses

- [ ] Список курсов загружается.
- [ ] Карточки курсов показывают название, описание, автора, навыки и картинку.
- [ ] Детальная страница курса открывается.
- [ ] Кнопки продолжения/прохождения ведут в правильный сценарий.
- [ ] Пустые или курсы без уроков отображаются без падения.

## Lessons

- [ ] Текстовый урок отображает контент.
- [ ] Видео урок показывает video/HLS состояние без ошибки.
- [ ] Attachments отображаются и открываются.
- [ ] Прогресс просмотра сохраняется.
- [ ] Завершение урока обновляет прогресс курса.
- [ ] Недоступный lesson manifest/progress не раскрывается чужому пользователю.

## Quiz

- [ ] Quiz загружает вопросы и варианты.
- [ ] Single-choice вопрос работает.
- [ ] Multiple-choice вопрос работает.
- [ ] Ошибочные ответы показывают результат без падения.
- [ ] Лимит попыток и блокировка отображаются, если настроены.
- [ ] Успешный quiz завершает урок.

## Teacher Analytics

- [ ] Summary cards загружаются.
- [ ] Список курсов в аналитике загружается.
- [ ] Список студентов в аналитике загружается.
- [ ] Фильтр/search не ломает таблицы.
- [ ] Пустая аналитика нового teacher отображается корректно.

## Vacancies

- [ ] Гость видит опубликованные вакансии.
- [ ] Поиск вакансий работает.
- [ ] Фильтр по навыкам работает.
- [ ] Фильтр по формату работы работает.
- [ ] Student видит matched vacancies.
- [ ] Employer видит свои вакансии.
- [ ] Черновики/закрытые вакансии не появляются в публичном списке.

## Vacancy Applications

- [ ] Student отправляет отклик один раз.
- [ ] Повторный отклик на ту же вакансию блокируется.
- [ ] Student видит список своих откликов.
- [ ] Student может отозвать pending/viewed отклик.
- [ ] Employer видит отклики только на свои вакансии.
- [ ] Employer меняет статус pending/viewed/accepted/rejected.
- [ ] Student получает обновление статуса.

## Notifications

- [ ] Колокольчик показывает количество непрочитанных.
- [ ] Student видит offers и vacancy application updates.
- [ ] Employer видит новые отклики и ответы на offers.
- [ ] Открытие уведомления помечает его прочитанным.
- [ ] Действия из уведомления обновляют статус.
- [ ] После действия счётчик уведомлений обновляется.

## Settings / Profile

- [ ] Profile открывается после login.
- [ ] Avatar загружается и отображается после обновления.
- [ ] Bio сохраняется.
- [ ] Settings открываются.
- [ ] Контакты сохраняются.
- [ ] Смена пароля работает с валидными данными.
- [ ] Ошибки формы показываются без падения страницы.

## Responsive

Проверить на ширинах:

- [ ] 360px: header, меню, карточки, формы, модальные окна, таблицы.
- [ ] 768px: планшетная сетка, карточки курсов, вакансии, профиль.
- [ ] 1440px: desktop layout, sidebar/course builder, analytics, vacancies.

На каждом breakpoint:

- [ ] Нет горизонтального скролла страницы, кроме специальных таблиц/heatmap.
- [ ] Текст не вылезает из кнопок и карточек.
- [ ] Модальные окна помещаются по высоте и скроллятся.
- [ ] Основные кнопки доступны без перекрытий.
- [ ] Изображения и аватары не растягиваются некорректно.
